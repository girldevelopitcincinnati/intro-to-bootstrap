## What is Bootstrap? According to Bootstrap, it's the most popular *frontend component library* for developing *responsive*, *mobile-first* projects on the web.
## What is Bootstrap? **Frontend component library** A collection of production-ready HTML/CSS/JavaScript components you can use with little configuration.
## What is Bootstrap? **Responsive** A Web design approach allowing a single webpage to be viewed in response to the size of the screen/web browser it's being viewed with.
## What is Bootstrap? **Mobile-first** Developing a webpage, beginning with your mobile design/layout, and gradually adjusting it so it looks good and behaves well on larger screen sizes.
## What we'll be covering - Setting up your project - The grid - Working with components - Using utility classes - Adding custom CSS

## Project setup There are a number of ways you can add Bootstrap to your project and each way has its pros and cons. The three most common ways are to: - download the source files - download the compiled files - use a CDN
## Project setup: Source files This is the kitchen sink option - it comes with *everything*. When you choose this option, you're getting uncompiled Sass files and ES6 Javascript - in addition to a bunch of other stuff. This includes the code and copy used to create Bootstrap's official website.
## Project setup: Source files Getting raw files like this is great for developers with their own builds set up (NPM, Gulp, etc.). It gives you the flexibility to do what you want with the code (theming, optimizing builds, etc.), but that flexibility does come with a learning curve.

## Project setup: Compiled files Downloading the compiled files gives you CSS and JS files that you can link to your project with little setup required. If you choose this option, just remember to link the *minified* CSS or JS file to your project.
## Project setup: CDN *CDN* stands for *Content Delivery Network*. Essentially, this means assets like styles or Javascript libraries can be served from a url online. Adding these assets is as simple as putting the link in the correct spot in your project. Using a CDN makes setting up your project a breeze, allowing you to start a project quickly.
## Project setup: CDN **Just remember**, every CDN added to your application can slow down the load time due to the multiple server requests required to run them. For educational or small projects, this shouldn't be a problem. For anything larger, you should consider using a different build option.
## Quick start template Bootstrap provides documentation if you choose to use any of these options (and more), but their "quick start" guide uses the CDN-approach. Navigate to Bootstrap's [Introduction](https://getbootstrap.com/docs/4.1/getting-started/introduction) section and we'll go over the starter template together.

## Adding your own styles Undoubtedly there will be times where Bootstrap won't have the component you need or the look you want. In cases like this, you might want to add your own HTML/CSS to the project.

Adding your own styles

To add your own styles, you'll want to create a new CSS* file and link it beneath the Bootstrap CSS files in the of your project.


*Note: If using the Bootstrap source files, you'll need to write your styles using a Sass preprocessor.
